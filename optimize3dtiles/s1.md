# 3D Tiles离线重组与Primitive合并模块设计文档

# 1. 项目目标

针对当前工程地质模型：

```text
Tile数量      1280
Primitive数量 2638
DrawCall      2638
```

存在：

* Tile过碎
* Primitive过碎
* DrawCall过高

的问题。

由于业务要求：

```text
模型必须全量加载
后续进行剖切分析
空间分析
射线分析
```

因此不采用LOD裁剪方案，而采用离线重组方案降低运行时开销。

---

# 2. 总体架构

```text
原始模型

tileset.json
│
├─ b3dm
├─ b3dm
├─ b3dm
└─ ...

        │

        ▼

模型预处理器

├─ Tileset重组模块
├─ Primitive合并模块

        │

        ▼

优化模型

optimized_tileset.json

├─ merged_001.b3dm
├─ merged_002.b3dm
├─ merged_003.b3dm
└─ ...
```

---

# 3. Tileset离线重组模块

## 3.1 设计目标

解决：

```text
root.children = 1280
```

导致的：

```text
Tile遍历开销
包围盒检测开销
Tile调度开销
```

问题。

---

## 3.2 输入

输入文件：

```text
tileset.json
```

读取内容：

```json
{
  "boundingVolume": {},
  "content": {},
  "transform": [],
  "children": []
}
```

---

## 3.3 数据提取

提取每个Tile：

```ts
interface TileInfo {

    id:string;

    uri:string;

    center:Cartesian3;

    boundingVolume:any;

    geometricError:number;

}
```

center计算来源：

```text
boundingVolume.box
```

中心点。

---

## 3.4 空间分组

采用：

```text
QuadTree
```

空间聚类。

分组策略：

```text
每8~16个Tile
组成一个Group
```

目标：

```text
1280 Tile

↓

100~200 Group
```

---

## 3.5 Group节点生成

生成父节点：

```json
{
    "boundingVolume": {},
    "geometricError": 1000,
    "refine": "ADD",
    "children": []
}
```

包围盒：

```text
所有子Tile包围盒并集
```

---

## 3.6 输出

输出：

```text
optimized_tileset.json
```

结构：

```text
Root

├─ Group001
│  ├─ Tile
│  ├─ Tile
│  └─ Tile

├─ Group002
│  ├─ Tile
│  └─ Tile

└─ ...
```

---

## 3.7 预期收益

```text
Tile数量

1280
↓

100~200
```

减少：

```text
包围盒计算
Tile遍历
Tile状态管理
```

---

# 4. Primitive合并模块

## 4.1 设计目标

解决：

```text
2638 Primitive
=
2638 DrawCall
```

问题。

---

## 4.2 输入

输入：

```text
*.b3dm
```

解析：

```text
b3dm
↓
glTF
↓
Mesh
↓
Primitive
```

---

## 4.3 数据结构

解析后：

```ts
interface PrimitiveInfo {

    materialId:string;

    textureId:string;

    positions:Float32Array;

    normals:Float32Array;

    uvs:Float32Array;

    indices:Uint32Array;

}
```

---

## 4.4 合并条件

满足：

```text
Material相同
Texture相同
Attribute布局相同
```

执行合并。

即：

```text
POSITION
NORMAL
TEXCOORD_0
```

结构一致。

---

## 4.5 顶点合并

执行：

```text
Vertex Buffer Merge
```

生成：

```text
Merged Position Buffer
Merged Normal Buffer
Merged UV Buffer
```

---

## 4.6 索引重映射

合并后：

```text
index += vertexOffset
```

重建Index Buffer。

---

## 4.7 Mesh重建

生成：

```text
Merged Mesh
```

替换原：

```text
N个Primitive
```

为：

```text
1个Primitive
```

---

## 4.8 b3dm重打包

重新生成：

```text
glb
```

然后：

```text
glb
↓

b3dm
```

输出：

```text
merged_xxx.b3dm
```

---

## 4.9 属性保留

必须保留：

```text
BatchId
FeatureId
PropertyTable
```

避免影响：

```text
拾取
属性查询
剖切分析
```

---

## 4.10 坐标处理

禁止：

```text
世界坐标合并
```

必须：

```text
保持局部坐标
```

避免：

```text
transform失效
```

---

# 5. 风险控制

## 不允许

```text
删除顶点
删除三角形
简化网格
LOD降级
```

因为：

```text
影响分析精度
```

---

## 允许

```text
Tile重组
Primitive合并
Buffer合并
索引重建
```

这些不会改变：

```text
几何形态
分析结果
```
